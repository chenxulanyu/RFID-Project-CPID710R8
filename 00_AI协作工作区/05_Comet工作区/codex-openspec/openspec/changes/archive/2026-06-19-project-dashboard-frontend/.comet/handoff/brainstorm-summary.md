# Brainstorm Summary

- Change: project-dashboard-frontend
- Date: 2026-06-19

## 已确认事实

- 展示端主布局选择 A：总览优先。
- 第一屏结构采用“项目标题/周期/当前日期 → KPI 指标 → 风险任务横条 → 任务明细表 → 轻量时间轴”。
- 该方向优先服务项目汇报和快速扫视，后续再下钻任务详情和时间轴。
- 本 change 纳入 `startDelayed` 延迟启动状态，用于区分“未来未开始”和“计划已开始但实际未开始”。
- 移动端竖屏策略选择强提示横屏：窄屏竖向打开时显示横屏引导，不展示完整主内容；横屏后展示完整仪表盘。
- 状态模型采用兼容式扩展：新增 `dashboardStatus: "finished" | "in-progress" | "start-delayed" | "not-started"`，保留旧 `elapsedDays` 字段。
- 用户强调视觉约束：文字和框体不能溢出，按钮、标签、表格单元格、时间轴条和移动端横屏提示都需要有响应式尺寸/换行/截断策略。

## 候选技术方案

- 首版 dashboard 使用 `web-app-foundation` 的 `ProjectProgressData` 服务契约，不直接读取 mock 数据。
- KPI 和风险清单由前端派生函数计算，避免与任务明细不一致。
- 时间轴先做轻量条形甘特，不引入复杂图表库。
