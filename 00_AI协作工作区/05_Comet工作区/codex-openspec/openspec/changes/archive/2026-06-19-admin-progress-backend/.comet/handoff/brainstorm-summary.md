# Brainstorm Summary

- Change: admin-progress-backend
- Date: 2026-06-19

## 已确认事实

- 首版后台数据保存采用浏览器本地保存，使用 localStorage 或等价本地持久化模拟真实后台保存。
- 后续 CloudBase 接入放到 `cloudbase-persistence` change，不在本 change 保存真实 CloudBase 密钥或接入云环境。
- 管理界面布局采用“左表右详情”：左侧任务列表用于扫描和选择，右侧详情表单用于编辑。
- 首版支持新增任务、编辑任务、软删除/归档任务，并支持恢复归档任务；不做物理删除。
- 完成比例采用混合模式：默认按日期和完成状态自动派生，但后台允许管理员手动覆盖完成百分比。
- 手动完成比例与实际结束日期冲突时，实际结束日期优先：存在实际结束日期即视为 100% 完成。
- 扣子编程部署由用户手动执行，不纳入本 change。

## 候选技术方案

- 本地 repository adapter：在现有 `ProjectRepository` 基础上增加 admin 读写接口，首版用 localStorage 保存项目元数据和任务输入。
- 管理页面：替换 `/admin` 占位页为左表右详情编辑界面，项目元数据与任务详情分区编辑。
- 展示端读取：`getProjectProgress()` 继续通过统一 repository/service 读取，使后台保存后的数据能被仪表盘读取。

## 关键取舍与风险

- localStorage 只适合首版本机演示和流程验证，不代表多人协作或云端数据持久化。
- 软删除/归档可以降低误删风险；展示端默认过滤已归档任务，后台保留已归档数据并提供恢复入口。
- 未实现认证权限，部署前必须在后续 change 中确认管理端访问控制。
