# fix-cloudbase-and-mobile-shell

## 问题

用户反馈部署后出现三个线上问题：

- 后台维护数据只在当前设备有效，其他设备仍看到默认内容。
- 手机端内容已横屏旋转，但左上角项目仪表盘/后台维护导航没有一起旋转。
- PC 端刷新后仍显示旧的 3 项任务。

## 根因

- 线上未拿到 CloudBase 前端认证配置时会静默回退到 `localStorage`，后台写入只保存在当前浏览器。
- `LandscapeGate` 包裹在页面组件内部，`App` 顶部导航位于旋转容器外。
- 已访问过旧版本的浏览器保存了 3 条任务的 `localStorage` 快照，优先级高于新的 31 条内置种子。

## 修复目标

- 生产默认使用 CloudBase 公共前端配置；测试环境仍可显式使用 local。
- CloudBase 和 localStorage 读取都以 31 条内置任务为基线，并叠加同 ID 的远端/本地覆盖。
- 顶部导航和页面内容一起进入横屏旋转容器。
