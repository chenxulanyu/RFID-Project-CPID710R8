# 设计

## 方案

将 `web/src/data/cpid710r8Mock.ts` 的默认任务数组从 3 条样例扩展为 Excel 进度明细。数据层保留 31 条明细任务，空白编号行继承上一个里程碑编号，因此 M5 等里程碑下的多条项目内容/进度名称不会丢失。仪表盘 KPI 的任务总数按唯一里程碑编号统计为 20。

## 范围

- 不修改 `.coze`、根 `package.json`、`serve.mjs` 等扣子部署适配文件。
- 不改变 CloudBase schema 和 repository 接口。
- 本次按用户确认的“20 个任务”理解为 20 个编号里程碑；Excel 中编号为空的展开明细纳入明细表和时间轴，但不重复计入 KPI 任务总数。
- 移动端竖屏不再展示提示页，改为在网页内部旋转仪表盘内容，以横屏布局显示。

## 验证

- `getProjectProgress()` 默认读取应返回 31 条明细任务。
- 唯一任务里程碑编号应覆盖 M1-M20，共 20 个。
- M5 应包含 3 条明细任务，保留 `V1.0 EVB PCB` 与 `V1.0 PCBA打样` 两个项目内容。
- 仪表盘 KPI 的 `totalTasks` 应按唯一里程碑编号统计为 20。
- 移动端竖屏 CSS 应隐藏提示页并旋转实际内容。
- CloudBase 返回旧结构或缺少必要任务字段时，前台读取应回退到内置完整种子数据。
