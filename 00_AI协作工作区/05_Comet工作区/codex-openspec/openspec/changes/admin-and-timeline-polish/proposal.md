# admin-and-timeline-polish

## Problem

部署后存在六个用户可感知的缺陷：

1. **新任务保存第一次报错**：新增任务首次保存提示"没有记录被更新"，第二次才成功。根因是 `saveDocument` 对 `set` 操作也检查了 `updated: 0`，而 `set` 是创建新文档，不存在"没有记录被更新"的概念。

2. **风险任务栏切换后显示异常**：从后台维护切回前台后，风险任务卡片高度被压缩，内容显示不全。之前加了滚动条但用户不想要滚动，要求固定高度。

3. **时间轴当前日期居中**：当前日期文本在时间轴区域居中显示，用户要求左对齐放在"计划时间轴"标题下方。

4. **归档任务列表间隔大**：已归档任务列表中存在明显空隙，排列不紧凑。

5. **缺少删除任务功能**：已归档任务无法删除，测试数据无法清理。

6. **项目时间范围应自动计算**：项目计划周期应根据所有任务的最早/最晚日期自动调整，同时允许用户手动扩大范围但不能缩小到比任务范围更窄。

## Root Cause Direction

- 问题1：`assertWriteSucceeded` 对 `set` 和 `update` 一视同仁检查 `updated: 0`，需拆分。
- 问题2：风险卡片 CSS 缺少固定高度和换行保护。
- 问题3：时间轴 CSS 用了 `justify-content: center`。
- 问题4：可能来自 seed 数据空任务或 `li` 元素间距。
- 问题5：`ProjectRepository` 接口缺少 `deleteTask`。
- 问题6：`dashboardMetrics` 时间范围直接用 `project.plannedStartDate/End`，未参考任务实际日期。

## Goals

- 修复新任务首次保存误报。
- 风险任务栏固定高度，不使用滚动。
- 时间轴当前日期左对齐在标题下方。
- 归档列表紧凑排列。
- 已归档任务可删除。
- 项目时间范围根据任务自动计算，用户可扩大但不能缩小到任务范围以内。

## Non-Goals

- 不修改扣子部署配置。
- 不修改 CloudBase 集合权限规则。
- 不调整既有 IPD 阶段目录。
