## Context

时间轴和任务明细都已经在展示完成比例，但不同容器下的文本排版会让百分号看起来像是丢失了。这个变更只针对显示层，不改动进度计算、时间轴布局算法或数据来源。

## Goals / Non-Goals

**Goals:**
- 让所有可见完成比例稳定显示为带 `%` 后缀的百分比字符串。
- 保持时间轴和任务表的现有视觉结构不变。
- 继续兼容桌面端与横屏移动端的密度布局。

**Non-Goals:**
- 不修改完成比例计算逻辑。
- 不修改任务排序、时间范围计算或甘特条宽度算法。
- 不引入新的图表库或新的数据字段。

## Decisions

### 1. 统一百分比文本格式的输出位置
进度标签统一通过现有格式化函数输出百分比字符串，避免不同组件各自处理而产生不一致。  
Alternatives considered:
- 在样式层拼接 `%`：可维护性差，且不解决语义一致性。
- 在数据层改造字段类型：范围过大，不符合本次修复目标。

### 2. 保持百分比标签的容器约束
既有容器宽度、换行和溢出控制继续沿用，只对显示文案做最小修正，避免引入新的视觉回归。  
Alternatives considered:
- 改大条形标签宽度：会挤压时间轴可视区域。
- 减少标签内容：会损失完成度信息。

## Risks / Trade-offs

- [Risk] 文本修复后仍可能因极窄容器被截断。→ Mitigation: 保持现有 `overflow` 和最小宽度策略，并通过测试覆盖关键视口。
- [Risk] 仅修显示层可能掩盖深层数据问题。→ Mitigation: 现阶段聚焦用户可见问题，数据链路保持现状，若后续复现再单独开修复。

## Migration Plan

1. 更新进度标签的渲染格式。
2. 补充回归测试覆盖百分比后缀显示。
3. 验证构建与页面显示。

Rollback:
- 如标签在极端宽度下仍不清晰，可恢复到当前格式化函数的调用点并调整容器样式，但不改变进度数据本身。

## Open Questions
