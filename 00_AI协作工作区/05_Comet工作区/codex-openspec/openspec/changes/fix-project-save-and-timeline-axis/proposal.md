# fix-project-save-and-timeline-axis

## Problem

线上后台仍然出现 `CloudBase保存失败：项目回读结果与提交内容不一致`，说明项目元数据写入后的回读校验仍存在误判或写入路径未覆盖真实 CloudBase 返回形态。同时，后台保存按钮位置和甘特图时间轴呈现与用户期望不一致。

## Root Cause Direction

项目保存逻辑会把 CloudBase 回读结果转换成 `Project` 后再比较。真实文档可能只稳定返回 `_id` 主键，而普通 `id` 字段可能缺失、旧值或不参与更新；当前比较将 `id` 也纳入保存确认，容易把“业务字段已写入成功”误判为回读不一致。需要让项目写入确认聚焦可编辑业务字段，并保留真实写入错误的失败提示。

## Goals

- 修复 CloudBase 项目信息保存的回读误判，尤其是修改计划日期后不能保存的问题。
- 后台布局恢复用户指定的位置：项目保存按钮在项目信息框左下角；任务保存按钮在任务信息框左下角，并与归档按钮对调。
- 甘特图移除条内开始/结束日期，计划时间轴区域只保留当前日期文本，去掉黑色当前日期长条。

## Non-Goals

- 不修改扣子部署配置。
- 不调整 CloudBase 集合结构或权限规则。
- 不重排既有 IPD 阶段目录。
