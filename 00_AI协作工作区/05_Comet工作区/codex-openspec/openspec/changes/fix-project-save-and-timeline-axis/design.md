# Design

## Save Confirmation

CloudBase 项目保存仍执行写后回读，但比较字段限定为后台可编辑的项目业务字段：`name`、`plannedStartDate`、`plannedEndDate`、`calendarMode`。`id` 由文档引用路径决定，不作为项目表单保存是否成功的判据。

为避免隐藏真实失败，仍保留：
- CloudBase 返回错误体时直接失败。
- CloudBase 返回 `updated: 0` 时直接提示检查集合写权限或文档归属。
- 回读业务字段连续两次不一致时失败。
- 回读为空时失败。

## Admin Layout

项目表单和任务表单继续独立保存。保存按钮使用左对齐动作行；任务动作行顺序调整为保存按钮在前，归档/恢复按钮在后。

## Timeline Display

计划时间轴的标题区域保留项目计划周期文字。时间轴内部不再展示起止日期刻度，也不再绘制当前日期黑色浮标；仅在时间轴顶部显示 `当前日期：YYYY-MM-DD`。任务条内只显示完成百分比。

## Verification

新增/更新测试覆盖：
- CloudBase 回读文档缺少普通 `id` 字段但业务字段已更新时，项目保存应成功。
- 后台按钮布局与顺序。
- 时间轴移除起止日期和当前日期浮标，只保留当前日期文本与百分比。
